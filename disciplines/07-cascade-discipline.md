# Discipline 7 — Cascade discipline (structural consistency)

When changing any concept, decision, primitive, or term in any doc: identify every other place it appears and update each in the same commit (or tightly-coupled sequence explicitly marked as completing the cascade). Changes propagate **up, down, and sideways**.

**Procedure-rigor for cascade-application is default-light + escalate-on-evidence**: default cascade-application = single sub-agent at xhigh effort with clear brief (per CLAUDE.md M1). Multi-sub-agent / Writer-Reviewer / cluster-execution methodology = escalation triggered by T1 architectural REVISION OR substantive new ARCH topic creation OR explicit user request — never default ceremony. Empirical grounding: 0 T1 across 14 cluster-executions per HANDOFF Notes 61-65; T2/T3 findings dominantly DOCUMENT-HYGIENE class catchable by single xhigh sub-agent.
