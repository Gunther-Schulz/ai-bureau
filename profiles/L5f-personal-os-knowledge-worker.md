Individual using framework as personal knowledge / information / task management system. Lighter framework usage than practitioner-shape: no professional-accountability-bearing output; no regulatory defensibility; no sparring requirements as load-bearing pillar. Personal use: writing, learning, organizing thoughts, tracking projects.

Per VISION's mention of personal-OS-shape as alternate framework shape. Validates that framework primitives generalize beyond practitioner-shape.

## What this profile is intended to EXEMPLIFY

This profile stress-tests:
- **Light-framework usage**: personal-OS-shape policies are MINIMAL compared to practitioner-shape
- **Audit granularity = "light"**: per shape policy; minimal events for memory/replay only; no external accountability
- **Sparring optional**: practitioner may not want sparring overhead for personal note-taking
- **Defensibility absent or trivial**: no regulatory/professional challenge scenario
- **Capacity-building still relevant**: personal learning + skill development through engagement
- **Single-user simplicity**: no multi-tenant; no governance complexity
- **Cross-shape primitive validation**: workspace, specialist, claim, etc. still work for personal-OS use case
