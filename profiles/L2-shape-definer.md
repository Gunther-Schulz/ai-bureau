## Identity + context

Architect who defines workspace shapes — the policy bundles configuring framework mechanisms (which mechanisms active, mandatory, default values per shape). Practitioner-shape, autonomous-business-shape, personal-OS-shape, federation-shape, hybrid-shape — and any new shape variants for archetypes the framework supports.

Distinct from L1 (specialist creator) — shape is policy bundle; specialist is bundled expertise. Distinct from L9 (shape catalog curator) — L2 creates new shapes; L9 maintains the catalog of shape variants over time.

## What this profile is intended to EXEMPLIFY

This profile stress-tests:
- **Shape DEFINITION boundary**: what's in shape's policy bundle vs out
- **Mechanism configuration semantics**: which mechanisms can shapes activate / mandate / set defaults for
- **Shape composability**: shape extends shape (DACH-planning extends planning-shape); shape composition rules
- **Shape variants vs distinct shapes**: when is X a variant of shape Y vs its own shape
- **Cross-shape policy consistency**: same mechanism configured differently across shapes
- **Shape versioning + migration**: shape update affecting deployed workspaces
- **License + IP** at shape level
- **Per-archetype shape design**: practitioner-shape (PBS marketed positioning) vs autonomous-business-shape (different value claims) vs personal-OS-shape (lighter framework usage)
