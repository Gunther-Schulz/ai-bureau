## Identity + context

Small architecture/engineering firm where multiple disciplines collaborate on building/infrastructure projects. Produces accountability-bearing technical artifacts: drawings, specifications, calculations, code-compliance reports, permit applications. Defensibility under building-code authority + structural-safety review + post-construction litigation.

Multi-disciplinary collaboration: architect leads design; structural engineer reviews load paths; MEP engineer specifies systems; landscape engineer specifies site work. Sequential review chains with discipline-specific authority.

Adds **multi-disciplinary collaboration + technical-artifact production** dimensions that other practitioner-shape profiles miss.

## What this profile is intended to EXEMPLIFY

This profile stress-tests:
- **Multi-disciplinary collaboration**: different professional disciplines (each with own license + accountability) within same workspace / project
- **Sequential review chains**: architect → structural review → MEP review → integrated check
- **Discipline-specific authority binding**: structural engineer signs structural; architect signs architectural; per-discipline boundaries
- **Technical artifact production**: drawings + specifications + calculations + reports — multi-format claim-bearing artifacts
- **Cross-artifact consistency**: drawings + specs + calculations must agree (consistency-as-defensibility)
- **Code-compliance documentation**: per-claim justification mapping to building code provisions
- **Post-construction defensibility**: claim defensibility extends through construction + occupancy + maintenance lifetime
- **Multi-jurisdictional projects**: cross-municipality / cross-state code variations
- **Per-discipline specialist activation**: architect-specialist + structural-engineer-specialist + MEP-engineer-specialist all active
- **Per-discipline workflow patterns**: each discipline has codified workflows; integration touch-points
