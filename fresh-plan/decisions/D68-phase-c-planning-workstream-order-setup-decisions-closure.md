## B. Setup decisions

**B.1 — Substrate library**: **Claude Agent SDK (Python)** as the Phase C real-wire substrate library. Replaces InProcessSubstrate (B2) for runtime execution; D63 §B Phase B baseline 8 implementation workstreams + B2b two-substrate parity stub stand unchanged. Per D36 §B.2 (Python primary; Claude Agent SDK is Python-native) + D12 framing.

**MS Agent Framework real-wire substrate deferred to Phase E multi-deployment validation** per A3 user resolution (2026-05-17). Rationale: B2b stub (per D41) already substantiates D17 substrate-neutrality structurally at Phase B; second real-wire substrate adds parity evidence but not new contract surface; Phase E is the natural home for cross-substrate validation alongside second-shape work. Phase C single-substrate keeps real-wire integration focus on the eight Phase B forward-bars + D21 deployability + D40 integrity-protocol extension point rather than diluting across two SDK ecosystems.

**B.2 — Persistence storage**: **JSONL append-only file** as the Phase C persistence-layer starting bar. Each event chain persists as one JSONL file per workspace; one event per line; append-only writes; full chain re-read at boot. Matches D40 §A "state IS derivable" property (chain replay reconstructs full state via D39); cheapest replay path; aligns with D10 append-only chain discipline at filesystem level.

Resolves D54 §D D-1 (classify_shape_change boot-integration — prior shape version derived from re-read chain + manifest comparison at boot) + D58 §D D-5 (snapshot caching strategy — re-derivation cost framing now concrete; deferred until query interface latency surfaces). Substantive alternatives + when to revisit:

- **SQLite**: better query interface for D40 §A `filter-by-*` / `state-at(n)` operations; revisit if Phase C real-wire chain sizes produce unacceptable replay latency (e.g., >100ms boot-time chain replay).
- **Postgres**: heavier dependency; Phase D PBS-Schulz natural home if multi-workspace federation or shared-storage scenarios surface.
- **Per-substrate plug-in persistence**: revisit if Phase E surfaces a second substrate (MSAF real-wire) with incompatible storage assumptions.

D40 §A query interface contract stands regardless of storage choice — JSONL impl provides the minimum (filter-by-actor / filter-by-work-unit / filter-by-payload-subtype / state-at / full-chain) via in-memory replay; SQLite swap would push these to indexed queries without contract change.

**B.3 — Adapter SDK libraries**: Phase C adapter workstreams (C3-C6) use the following library landscape, treated as **provisional 2026-05 ecosystem assumption with verify-at-workstream-start discipline**:

- **C3 (MCP client)**: official Anthropic MCP Python SDK (`mcp` package).
- **C4 (direct-API)**: per-API SDK as appropriate (e.g., `anthropic` for Claude direct; `openai` for OpenAI direct; HTTPX as fallback for HTTP-shaped APIs without dedicated SDK).
- **C5 (A2A peer)**: official A2A SDK (Python) — per D21 ecosystem framing; specific package name verified at C5 workstream start (A2A spec is still evolving 2026-05; ecosystem SDK landscape may have shifted).
- **C6 (MCP server)**: Anthropic MCP Python SDK server-side bindings (same package as C3, different surface).
- **C8 (AEGIS)**: AEGIS reference impl (specific Python package verified at C8 workstream start; AEGIS protocol described at D40 §B as canonical first integrity-protocol-extension target — its implementation maturity drives C8 timing).

**Verify-at-workstream-start discipline** (NEW; carries forward to Cref + Phase D / E planning): library claims locked here are 2026-05 ecosystem snapshots, not invariant decisions. Each adapter workstream re-verifies its SDK choice before committing impl work — confirms package exists + is maintained + matches D16 adapter-kind contract + matches the registered `protocol-or-transport` identifier (per D29 namespacing). Failure surfaces a verify-at-workstream-start finding; resolution either swaps SDK (one-line setup-decision amendment via clarification entry) or pivots to direct protocol impl (alternative locked at workstream-start).

## C. Closure criterion

Per D26 Phase C closure trigger ("A2A peer + MCP server demonstrably work in the reference workspace; at least one external peer interaction succeeds") + the eight Phase B forward-bars from D63 §C + the D40 §B integrity-protocol extension point. **(b) MCP server external invocation succeeds.** C6 MCP server adapter exposes loaded specialists' skills as MCP tools; at least one external MCP client (Claude Code / Claude Desktop / test harness) successfully invokes a specialist skill via MCP tool-call. Validates D21 generalization (workspace-as-MCP-server).

**(c) Real-wire RAG-via-MCP end-to-end.** C7 real-wire RAG specialist (replacing B7 stub) invokes a real MCP client adapter (C3) against a real MCP server (e.g., a bauleitplanung-corpus retrieval server stub or equivalent) end-to-end; retrieved content threads through `handle_skill` body without exception; emitted action event carries real (not stubbed) `result` payload. Closes B7 stub → real-wire transition (per D63 §E Phase C scope handoff item 3).

**(d) Persistence survives workspace restart with D54 + D58 active.** C2 persistence-layer-bound workspace reboots; event chain re-read from JSONL storage; state re-derived per D39 produces identical workspace state. Activates D54 §B.2 `classify_shape_change` boot integration (resolves D54 §D D-1 — prior shape version derived from re-read chain at boot; `WorkspaceBootError(category="shape-migration-unsafe")` raises on breaking version-bump). **(e) D48 §B.1 AdapterCallError raised under real-wire failure conditions.** At least one C3 / C4 / C5 / C6 adapter raises `AdapterCallError` from a real-wire `call()` failure across each starter category — `transport` (network unreachable / DNS failure), `auth` (credential rejected / token expired), `timeout` (real network timeout), `protocol-error` (malformed JSON-RPC / unexpected response shape), `upstream-error` (upstream server 5xx). Per D45 triad: detection (real-wire raise) + surface (structured fields per D48 §B.1) + recovery (caller / shape policy decides per category). Per-protocol category vocabulary extensions (per D48 §D D-3) registered as discovered.

**(f) D50 §B.1 SkillExecutionError raised under real-wire specialist failure conditions.** At least one C7 real-wire specialist raises `SkillExecutionError` from a real-wire `handle_skill` failure across each starter category — `domain-error` (input validation rejection), `external-dependency-error` (composed `AdapterCallError` from `adapter.call` inside `handle_skill` per D50 §B.2 + D50 §D D-5), `skill-execution` (genuine skill-body runtime error). Per D45 triad applied to specialist runtime. The D50 §D D-5 composition contract (wrap-vs-propagate AdapterCallError-inside-handle_skill) resolves to a concrete precedent during this workstream — first real-wire specialist establishes the chosen pattern.

**(g) AEGIS integrity-chain verification round-trip.** C8 AEGIS-shaped integrity-protocol extension ships; event chain is integrity-checkable per D40 §B (SHA-256 hash chain + Ed25519 signing + JCS canonicalization per D40 §B canonical-first-example); at least one chain-verification round-trip succeeds (chain emitted → persisted → re-read → integrity verification passes); tampered-chain detection (chain modified → integrity check fails with structured diagnostic) is the standard test pattern demonstrating the property — implementation discretion on inclusion as test fixture. Operationalizes D10 "integrity-checkable" promise + D40 §B extension point + EU AI Act Article 12 audit-record reconstruction (per external-trigger checkpoint 2026-08-02).
