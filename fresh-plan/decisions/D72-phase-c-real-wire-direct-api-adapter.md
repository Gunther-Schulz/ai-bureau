## B. Triad applied per path

### B.1 — Adapter `call()` real-wire failure (Phase C real-wire forward-bar ACTIVATION for HTTP; resolves D48 §D D-3 for HTTP transport)

| Triad element | Lock |
|---|---|
| **Detection** | `RealWireDirectAPIAdapter.call()` synchronously drives `httpx.AsyncClient.post` via `asyncio.run`. Detection points: (a) `httpx` module import failure → mapped to `transport`; (b) `_session_factory` invocation OR `AsyncClient` constructor raising → propagated up through asyncio.run to the outer exception mapper; (c) `client.post()` raising `httpx.ConnectError` / `httpx.NetworkError` → `transport`; (d) `httpx.TimeoutException` (covers ConnectTimeout + ReadTimeout + WriteTimeout + PoolTimeout subclasses) → `timeout`; (e) `response.raise_for_status()` raising `httpx.HTTPStatusError` with `response.status_code` in {401, 403, 407} → `auth` (NATIVE HTTP mapping per RFC 7235 + 7239); (f) `httpx.HTTPStatusError` with other 4xx (400/404/405/409/422/429/etc.) OR 5xx (500/502/503/etc.) → `upstream-error`; (g) `httpx.RemoteProtocolError` / `httpx.DecodingError` from transport-level framing/encoding errors → `protocol-error`; (h) `response.json()` raising `ValueError` (subclass: `json.JSONDecodeError`) when content-type declared JSON but body malformed → `protocol-error` (server returned 2xx but body didn't conform to declared content-type); (i) raw `OSError` from MockTransport handler / lower-level socket → `transport`; (j) any other Exception → `unknown` (catch-all per D48 §B.1 starter vocabulary terminal branch). Call-lifecycle raise-point (per D48 §D D-1 resolution + D71 §B.1 carryforward): action event emitted at method entry BEFORE wire round-trip; AdapterCallError can raise from anywhere downstream (mid-wire after client construction OR during POST OR during response interpretation). |
| **Surface** | `AdapterCallError` raised carrying structured fields: `adapter_id` (the adapter spec's id; `direct-api-realwire-adapter` for the 0.2.0 provision); `call_target` (the `tool_name` parameter — interpreted as the URL path appended to `base_url`); `category` (string; one of `transport` / `auth` / `timeout` / `protocol-error` / `upstream-error` / `unknown` per starter vocabulary); `detail` (dict carrying HTTP-specific structured context — `reason` always; `status-code` for HTTPStatusError-derived categories; `read-timeout-seconds` for timeout). Original exception chained via Python's `__cause__` attribute (`raise ... from exc`) — caller sees full traceback chain for diagnostic visibility. When AdapterCallError fires from inside a specialist's `on_event` (subscriber-dispatch path per D37 + D44 + D47), captured per D47 §B.1 into substrate's `_subscriber_failures` and aggregated as `SubscriberDispatchError` after outer drain — composition contract preserved per D48 §B.1 + D71 §B.1 carryforward. |
| **Recovery** | Caller catches `AdapterCallError`, inspects `.category`, decides per-category recovery (retry / fail-fast / circuit-breaker / fall-through / credential-refresh-on-auth). Framework does NOT mandate retry semantics per D48 §B.1 caller-policy framing. Chain integrity preserved per D10 + D39 + D40 §A: the `action` event emitted PRE-WIRE stands in the chain regardless of `call()` outcome (per D48 §D D-5 resolution at D71 §B.1; D72 §B.1 carries this pattern — emit-before-call chosen so partial-success paths leave intent-recorded action event in chain). Per-category caller strategies (transport: retry-with-backoff; timeout: retry or surface; protocol-error: surface as bug + abort; upstream-error: caller-specific; **auth: refresh-credentials + retry** — first natively-discriminable in C4); the auth-recovery strategy is Phase D / per-API extension territory per D-2 below. |
