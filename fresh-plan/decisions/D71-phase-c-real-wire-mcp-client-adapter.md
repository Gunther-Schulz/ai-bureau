## Rationale

D68 §A locked C3 as the third Phase C workstream + dependent on C1 + C2 (per dependency table). D68 §B.3 named the MCP Python SDK as the Phase C C3 library with verify-at-workstream-start discipline. D68 §C closure item (e) named D48 §B.1 raised under real-wire failure conditions per starter category vocabulary as a closure criterion. D71 instantiates all three — installs `mcp>=1.0`, verifies the API, lands the RealWireMCPClientAdapter impl, surfaces the first Phase C real-wire adapter category vocabulary per D45's standing triad requirement.

The choice to **add a new adapter alongside MCPToolAdapter + DirectAPIAdapter** (rather than replace MCPToolAdapter) honors D41 two-substrate parity + the parallel growth pattern set by Phase B + D69's substrate-alongside precedent. Each adapter impl is its own extension at `extensions/<id>-ext/<version>/`; the registry dispatch (`_ADAPTER_CLASSES`) selects by protocol-or-transport identifier at boot time. Future Phase C workstreams (C4 direct-API real-wire; C5 A2A peer; C6 MCP server-side) follow the same pattern.

The choice to **register a NEW protocol-or-transport identifier `mcp-client-realwire` (rather than re-use 0.1.0's `mcp-client`)** preserves the 0.1.0 stub path for back-compat + lets workspaces opt into real-wire by binding the 0.2.0 provision explicitly. The alternative (re-using `mcp-client` and replacing the registry entry to point at RealWireMCPClientAdapter) would break the 6 existing test_adapter.py tests that expect MCPToolAdapter stub behavior. Per D41 + D69 substrate-alongside spirit: real-wire impls add capability, they don't subtract.

The choice to **emit the `action` event BEFORE the wire round-trip** (resolves D48 §D D-5) honors the D5 §I3 accountability-anchor framing + D10 chain integrity: the intent to call a tool is recorded in the chain regardless of whether the call succeeds. Partial-success and failure paths preserve the action event; caller may emit a subsequent `state-change` event to record outcome. The alternative (emit-after-success) would lose the intent record on failure — a worse audit-trail property.
