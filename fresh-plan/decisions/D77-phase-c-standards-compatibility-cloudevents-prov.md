## A. Scope of cluster

ONE module + ONE adapter class + ONE extension + ONE pyproject dep + ZERO new FAILURE_CATEGORIES + ZERO event envelope changes. Honest cluster cardinality: **2 paths** — (a) CloudEvents conversion (single fresh-plan event → CloudEvent 1.0) + (b) PROV-JSON conversion + export (event chain → PROV-JSON dict OR file). NON-BREAKING design (D77 §A) is the load-bearing decision for the cluster shape — a breaking envelope rename would cascade through 6 payload schemas + every test fixture + every event-shape assumption across c1-c8 (332+ tests).

In scope:

- **NEW module** at `fresh-plan/impl/src/fresh_plan/runtime/standards_export.py` (300 LOC including docstring + 3 conversion functions + 1 validation helper):
  - `to_cloudevents(event: dict, *, workspace_id: Optional[str] = None) → cloudevents.v1.http.CloudEvent` — CloudEvents 1.0 mapping per CNCF spec (id ← event.id; source ← `workspace://<workspace-id>`; type ← payload-subtype; specversion='1.0'; datacontenttype='application/json'; time ← timestamp; data ← payload). Raises ValueError on malformed event (no silent substitution).
  - `event_chain_to_prov_json(events: list[dict], workspace_id: str) → dict` — W3C PROV-JSON shape: prefix block (prov namespace + fresh-plan namespace) + activity map (per-event prov:Activity with prov:startTime + prov:type) + agent map (per unique actor.id prov:Agent + workspace authority prov:Agent) + wasAttributedTo map (per (actor, event) + per (workspace, event) relations). Empty chain yields document with 0 Activities + workspace Agent anchored. Raises ValueError on malformed event OR empty workspace_id.
  - `write_prov_json(events, workspace_id, output_path) → dict` — IO wrapper: invokes converter + persists JSON to disk (indent=2, sort_keys=True for determinism) + returns dict for caller introspection. Raises OSError on IO failure.
  - `_validate_event_envelope(event, *, context) → None` — internal helper per D77 §B.2 Detection: raises ValueError on missing required envelope field (id / timestamp / payload-subtype / actors / payload) OR empty actors list.

## B. Triad applied per path

### B.1 — CloudEvents conversion (NEW shape per D77; per D24 standards-compat tracker)

| Triad element | Lock |
|---|---|
| **Detection** | At conversion time — `to_cloudevents(event, workspace_id=None)` calls `_validate_event_envelope` which raises `ValueError` when the event is not a dict OR is missing any required envelope field (id / timestamp / payload-subtype / actors / payload) OR has an empty actors list. CloudEvents required attributes (id / source / specversion / type) are MAPPED from fresh-plan envelope; source falls back to `workspace://unknown` when `workspace_id` is omitted (CloudEvents spec REQUIRES source; absent workspace context still produces a valid CloudEvent). |
| **Surface** | Direct callers receive `ValueError` with structured message naming the missing field(s) + offending event id (when available). When the conversion is invoked through `ProvJsonExportAdapter.call`, the ValueError is re-wrapped as `AdapterCallError(category='protocol-error')` per D48 §D D-3 starter category vocabulary REUSE. Pure-converter callers can match on the ValueError directly (test pattern). |
| **Recovery** | Caller decides per call site. Direct usage: caller inspects ValueError message + corrects the event payload upstream OR aborts the export pipeline. Adapter usage: AdapterCallError propagates through the standard adapter-call surface; caller catches per D48 §B.1 forward-bar. No silent substitution — a malformed event cannot produce a downstream CloudEvent / PROV record with missing canonical fields. |

### B.2 — PROV-JSON conversion + export-adapter IO (NEW EXPORT-ONLY adapter sub-shape per D77; activates D24 standards-compat tracker for W3C PROV-DM impl-level milestone)

| Triad element | Lock |
|---|---|
| **Detection** | Three classes of detection point: (1) **At conversion time** — `event_chain_to_prov_json(events, workspace_id)` iterates events; each event passes through `_validate_event_envelope` which raises `ValueError(subcase=missing-required-field, event-id=<id>)` on missing required envelope field; an actor entry without `id` field raises `ValueError(subcase=actor-without-id, event-id=<id>)`; empty `workspace_id` raises `ValueError(subcase=empty-workspace-id)`. (2) **At file IO time** — `write_prov_json(events, workspace_id, output_path)` calls `output_path.write_text(...)` which raises `OSError` (PermissionError / IsADirectoryError / FileNotFoundError / OSError per platform) on filesystem failure (readonly dir, permission denied, disk full, missing parent dir). (3) **At adapter time** — `ProvJsonExportAdapter.call('export', {output-path?})` is the wrapping surface: pre-export `action` event emitted via `_emit_action` per D71 pre-wire convention; then converter + IO invoked; failures caught + wrapped per category. |
| **Surface** | Pure-converter callers see `ValueError` (conversion) OR `OSError` (IO). Adapter callers see `AdapterCallError` per D48 §D D-3 starter category vocabulary REUSE: `category='protocol-error'` for ValueError (malformed event envelope) with `detail={reason: 'PROV-JSON conversion: malformed event envelope', error: <str>}`; `category='transport'` for OSError (file IO failure) with `detail={reason: 'PROV-JSON file write failed', output-path: <str>, error: <str>}`; `category='unknown'` catch-all with `detail={reason: 'PROV-JSON export: unexpected error', error: <str>, error-type: <type-name>}`. Action event remains in the workspace chain regardless of subsequent failure (export-side-effect transparency); the export payload itself is not produced on failure. |
| **Recovery** | Caller catches `AdapterCallError` per D48 §B.1 forward-bar + dispatches on category. `protocol-error` indicates upstream bug (chain has a malformed event); operator inspects per-event audit + repairs OR aborts. `transport` indicates filesystem environment issue (permission / disk); operator corrects OR retries with a different output-path. `unknown` indicates an unexpected exception (rare); operator surfaces detail for forensic analysis. The pre-export action event recorded in the chain enables post-hoc audit of all export attempts (success + failure) per D71 pre-wire convention. Per scope-cut C12: in-process tests only via pytest tmp_path; production reach (cloud storage / regulator submission endpoints) deferred to Phase D pioneer-instance deployments. |
